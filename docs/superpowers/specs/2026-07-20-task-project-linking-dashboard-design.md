# 任務專案關聯與儀表板進行中工作設計

## 目標

讓產品工程師可以同時管理日常雜項與專案工作；在儀表板快速看出正在推進的事項，而每一筆任務可保留完整的目前進度與背景。

## 範圍

本次涵蓋：

1. 新增任務時可指定日常雜項或一個進行中的專案。
2. 將既有任務備註在介面明確標示為「目前進度／備註」。備註採長文字、展開後編輯，不顯示在儀表板。
3. 儀表板新增獨立的「進行中」任務區塊及對應 KPI。
4. 專案詳情新增與該專案關聯的任務列表。

本次不新增標籤、等待狀態、提醒機制、任務類型或修改既有備份格式以外的資料遷移。

## 資料設計與相容性

既有 `Task` 已有 `projectId` 與可選的 `notes`，因此不新增資料欄位。

- 新任務必須帶 `projectId`。
- 「日常行政／雜項」繼續使用既有的 `default-misc-project`。
- 任務的 `notes` 保留原始值；介面只更新名稱與說明為「目前進度／備註」。
- 舊任務無須遷移，會保持原先的專案歸屬。
- 任務狀態為「進行中」時，才會被納入儀表板的新區塊；父任務依既有有效狀態規則判定。

## 任務頁與建立流程

「新增任務」表單加入專案選擇器：

- 預設為日常行政／雜項，維持快速輸入習慣。
- 選項包含日常行政／雜項與狀態為進行中的專案。
- 任務名稱、到期日、急件操作保留。

任務列既有的備註按鈕與展開文字區維持行為，但按鈕提供「目前進度／備註」語意，輸入區提示使用者記錄目前情況、阻塞點或下一步。

## 儀表板

在既有待辦與專案概覽之外，新增「進行中」區塊：

- 只顯示狀態為「進行中」的任務。
- 依有效到期日由近到遠排序，未排日期放最後。
- 每列呈現任務名稱、所屬專案（雜項亦顯示）、到期日與急件標記。
- 可直接將任務標示完成；點擊任務可導向待辦頁以編輯目前進度。
- KPI 新增進行中任務數量；既有逾期、今日、本週與進行中專案資訊維持。

## 專案詳情

在專案詳情中加入「關聯任務」區塊：

- 顯示 `projectId` 相同的任務，包含進行中與待辦，已完成任務可依既有顯示慣例折疊或隱藏。
- 任務列沿用現有 `TaskRow` 的狀態、日期、急件與目前進度／備註操作，避免產生兩套不同編輯介面。

## 錯誤處理與邊界

- 若舊資料引用已刪除的專案，任務仍保留並以「未歸屬專案」顯示，不阻斷儀表板。
- 專案選擇器只列出進行中的專案，降低誤掛到已完成或取消專案的機率。
- 目前進度／備註為選填，不影響任務的完成與日期邏輯。

## 驗證

1. 建立雜項任務與指定專案任務，確認資料儲存並在重新整理後仍存在。
2. 將任務改為進行中，確認 KPI 與「進行中」區塊正確更新、排序與完成操作正常。
3. 在任務填寫目前進度／備註，確認展開、保存及既有備註資料顯示正常。
4. 開啟專案詳情，確認只顯示該專案的關聯任務。
5. 執行 TypeScript 編譯與正式建置。
